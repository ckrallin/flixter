Lesson 12: Adding a Sticky Footer

Before we jump into fun database stuff, let's quickly add a sticky footer to our application. Once we get our sticky footer squared away, we will have the foundation of our web application built.

Footers are tricky to get perfect at first. They are tricky to add because a footer needs to act differently depending on the situation. If we don't have a page filled up with content, our footer should snap to the bottom of the screen. But if our page does have enough content to fill the screen, we want the footer to be positioned at the very bottom of the screen, below the content.

So how do we do this? Google is our friend here: If you do a Google search for sticky footer css, one of the top results is titled Sticky Footer, Five Ways | CSS Tricks found here. If we scroll down the page, we eventually go to a section called There is flexbox.

Up to this point, we have been using the built-in grid system of Bootstrap to lay out our DOM elements. It turns out, though, that CSS has a native way to lay out DOM elements responsively so that they change size and move depending on the screen size. The feature is called Flexbox. Flexbox is very powerful, built into CSS, and allows us to dynamically align DOM elements in rows and columns with containers. You can learn more about Flexbox here.

For our purposes right now, we only need to worry about the code in the article article we found earlier.

In the article we found, it shows us code snippets for implementing the Flexbox to create a sticky footer. We need a div container which surrounds all the content except the footer and some CSS for our body and html tags.

Let's jump into app/views/layouts/application.html.erb and add the code now. 

<!DOCTYPE html>
<html>
  <head>
    <title>Flixter</title>
    <%= csrf_meta_tags %>
    <%= stylesheet_link_tag    "application", media: "all", "data-turbolinks-track" => true %>
    <%= javascript_include_tag "application", "data-turbolinks-track" => true %>
  </head>
  <body>
    <div id="flex-content-container">
      <nav class="navbar navbar-inverse bg-inverse">
        <button class="navbar-toggler hidden-lg-up" type="button" data-toggle="collapse" data-target="#navbarResponsive" aria-controls="navbarResponsive" aria-expanded="false" aria-label="Toggle navigation"></button>
        <div class="collapse navbar-toggleable-md" id="navbarResponsive">
          <%= link_to 'Flixter', root_url, class: 'navbar-brand' %>
          <ul class="nav navbar-nav">
            <li class="nav-item active">
              <a class="nav-link" href="#">Home <span class="sr-only">(current)</span></a>
            </li>
            <li class="nav-item">
              <a class="nav-link" href="#">Link</a>
            </li>
            <li class="nav-item">
              <a class="nav-link" href="#">Link</a>
            </li>
          </ul>
          <ul class="nav navbar-nav float-lg-right">
            <% if user_signed_in? %>
              <li class="nav-item"><%= link_to 'Sign out', destroy_user_session_path, method: :delete %></li>
            <% else %>
              <li class="nav-item"><%= link_to 'Sign in', new_user_session_path %></li>
              <li class="nav-item"><%= link_to 'Sign up', new_user_registration_path %></li>
            <% end %>
          </ul>
        </div>
      </nav>

      <% if notice.present? %>
        <p class="alert alert-info"><%= notice %></p>
      <% end %>

      <% if alert.present? %>
        <p class="alert alert-danger"><%= alert %></p>
      <% end %>

      <%= yield %>
    </div>

    <div id="footer">
      <div class="container">
        <b>Footer Here</b>
        <br />
        <%= link_to 'Link', '#' %>
      </div>
    </div>
  </body>
</html>

Next we need to include the CSS that will activate our sticky footer.

In the past we've dumped every single line of CSS in our master.scss file, but as our application gets bigger and bigger, it makes sense to move our CSS for different components of the web application into corresponding CSS files. Let's make a new file called app/assets/stylesheets/footer.scss. This file we'll reserve for CSS about the sticky footer. In that file add the following content, which comes from the site we found, but with a few modifications to match our HTML: 

html {
  height: 100%;
}

body {
  min-height: 100%;
  display: flex;
  flex-direction: column;
}

#flex-content-container {
  flex: 1;
}

Save this file and refresh the page.

Now let's make sure the footer moves dynamically, relative to where it is in our code, and keeps max-width with our browser. I also want the footer to be black and semi-transparent. Open your footer.scss and add this code to set the background: 

html {
  height: 100%;
}

body {
  min-height: 100%;
  display: flex;
  flex-direction: column;
}

#flex-content-container {
  flex: 1;
}

#footer {
  position: relative;
  width: 100%;
  min-height: 100px;
  background-color: rgba(36, 32, 33, 0.9);
}

Save the file and refresh the page.

Finally, let's adjust the styles inside the footer so it looks a little better. Since we can't see the text, let's make the text white. Open up footer.scss and make it look like this: 

#footer {
  position: relative;
  width: 100%;
  min-height: 100px;
  background-color: rgba(36, 32, 33, 0.9);
  color: white;
}

Nice. Now let's make the link white as well. Add this to the bottom of the footer.scss file. 

#footer a {
  color: white;
}

Save the file and refresh the page. It's starting to look better. Next, adjust the font family to be Helvetica inside the footer. 

#footer {
  position: relative;
  width: 100%;
  min-height: 100px;
  background-color: rgba(36, 32, 33, 0.9);
  color: white;
  font-family: "Helvetica Neue", Helvetica;
}

Save the file and refresh the page. The links would look better if the weight was super thin, so let's specify the font weight: 

#footer a {
  color: white;
  font-weight: 100;
}

And finally, let's add 10 pixels of padding to the top and bottom of the footer, but have a zero padding left and right: 

#footer {
  position: relative;
  width: 100%;
  min-height: 100px;
  background-color: rgba(36, 32, 33, 0.9);
  color: white;
  font-family: "Helvetica Neue", Helvetica;
  padding: 10px 0px;
}

The last thing we can do to space out our web application is to push the footer down a bit to make sure it doesn't ever overlap any of our content: 

#footer {
  position: relative;
  width: 100%;
  min-height: 100px;
  background-color: rgba(36, 32, 33, 0.9);
  color: white;
  font-family: "Helvetica Neue", Helvetica;
  padding: 10px 0px;
  margin-top: 30px;
}

Save the file and refresh the page. Our footer looks awesome! Building a sticky footer was a bit tricky to get right, but it's worth it. Everyone likes a good footer.
